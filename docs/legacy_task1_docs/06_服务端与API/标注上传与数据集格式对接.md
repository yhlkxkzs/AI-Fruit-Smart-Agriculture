# 标注上传与数据集格式对接

## 一、接口

- **路径**: `POST /annotation/upload`（与 MobileNet API 同服务，同端口）
- **Content-Type**: `multipart/form-data`
- **表单字段**:
  - `image`: 文件（本次校准对应的图片，字段名也可为 `file`）
  - `annotation`: 字符串（UTF-8），为下方「annotation JSON」的字符串形式

## 二、annotation JSON 格式（App 校准表单）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fruit_type | string | 是 | 果实种类：苹果、梨、桃、葡萄、柑橘、香蕉、草莓、西瓜、其他 |
| leaf_condition | string | 否 | 叶片状态：健康、黄化、枯斑、卷曲、缺失、其他 |
| maturity | string | 是 | 成熟度：未熟、转色、成熟、过熟、未知 |
| has_disease | boolean | 是 | 是否有病害（含虫害） |
| disease_name | string | 否 | 病害/虫害名称 |
| disease_severity | string | 否 | 病害程度：无、轻度、中度、重度 |
| image_type | string | 是 | 图片类型：真实照片、油画、素描、水彩、卡通、其他 |
| multiple_fruits | boolean | 是 | 是否多种水果 |
| notes | string | 否 | 备注 |
| client_time | string | 否 | 客户端时间 ISO8601 |
| detection_id | string | 否 | App 端识别/校准 ID |

## 三、服务端实现说明（本仓库已实现）

- 解析 `multipart/form-data`，取出 `image`（或 `file`）和 `annotation`。
- 将图片存到 **datatemp** 目录下，按**水果种类（英文）**分子目录，文件名唯一：
  - 根目录：`task1_fruit_classification/datatemp/`
  - 子目录：`<fruit_type 英文>/`，如 `apple/`、`pear/`、`peach/`、`grape/`、`citrus/`、`banana/`、`strawberry/`、`watermelon/`、`other/`
  - 图片：`<fruit_type>/annotation_<时间>_<uuid>.jpg`（可扩充，每次上传新文件不覆盖）
  - 标注：`<fruit_type>/json/annotation_<时间>_<uuid>.json`，格式与现有 dataset 的 json 一致（含 info、images、annotations、categories），并保留校准字段（fruit_type、leaf_condition、maturity、has_disease 等）于 `images[0]` 中。
- 成功返回 HTTP 200，Body：`{"success": true, "annotation_id": "<英文类名>_<唯一 stem>"}`。
- 失败返回 4xx/5xx，Body：`{"success": false, "error": "..."}`。

## 四、果实种类中文 → 英文目录名

| 中文 | 英文目录名 |
|------|------------|
| 苹果 | apple |
| 梨 | pear |
| 桃 | peach |
| 葡萄 | grape |
| 柑橘 | citrus |
| 香蕉 | banana |
| 草莓 | strawberry |
| 西瓜 | watermelon |
| 其他 | other |

## 六、落盘结构（可扩充）

```
datatemp/
├── apple/
│   ├── annotation_20260310_120000_a1b2c3d4.jpg
│   └── json/
│       └── annotation_20260310_120000_a1b2c3d4.json
├── pear/
│   ├── annotation_20260310_120100_e5f6g7h8.jpg
│   └── json/
│       └── annotation_20260310_120100_e5f6g7h8.json
└── ...
```

新图片不断上传时，仅新增文件，不覆盖已有文件，便于数据集持续扩充。
